## 背景与目标
很多同学有“把公开视频保存到本地”的需求，但平台往往存在：
- 不支持下载
- 操作不便（不支持批量、限制多、清晰度/格式受限）
- 跨设备不顺（手机端尤其明显）

因此希望做一个**万能视频下载网站**：更快速、随时随地、跨平台下载视频，手机上也能下载。

> 备注：项目更偏学习与工程实践。请尊重版权与平台规则，下载登录/会员内容有封号风险。

## 目标用户与核心场景
- **学生/自媒体/内容整理者**：把教程/资料视频离线保存
- **移动端用户**：手机浏览器直接保存到本地
- **批量整理**：一次性处理多条链接并查看进度

## MVP（阶段2）范围
MVP 聚焦“下载体验”，先不引入数据库，尽量轻量。

### 必须具备
- **单链接下载**：粘贴一个 URL 创建任务
- **批量下载**：多行 URL 创建队列任务
- **实时进度**：任务状态 + 进度（百分比/速度/ETA）可视化
- **临时存储与自动清理**：文件保留有限时间，到期删除

### 明确不做/默认关闭
- **cookies/登录下载**：默认关闭，通过管理员开关启用（降低风险）
- **支付/会员**：MVP 只做 UI 占位与引导文案，不接入真实支付
- **数据库**：暂不做用户系统、历史记录、配额持久化

## 风险与约束
- **版权与平台风控**：必须在 UI 明显位置提示；功能层面默认关闭 cookies 能力
- **资源与滥用**：需要基本限流（单 IP 并发、单次 URL 数、TTL 清理）
- **平台适配复杂**：不自研解析，统一使用 `yt-dlp` 封装，避免维护与合规风险扩散

## 未来扩展方向（非 MVP）
- 格式/清晰度选择（formats 列表）
- 音频提取、字幕下载与翻译
- 视频总结（需要额外 AI/转写能力）
- 付费能力与配额体系（引入数据库后做）
