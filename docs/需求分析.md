## 背景与目标
很多同学有“把公开视频保存到本地”的需求，但平台往往存在：
- 不支持下载
- 操作不便（不支持批量、限制多、清晰度/格式受限）
- 跨设备不顺（手机端尤其明显）

因此希望做一个**万能视频下载网站**：更快速、随时随地、跨平台下载视频，手机上也能下载。

> 备注：项目更偏学习与工程实践。请尊重版权与平台规则，下载登录/会员内容有封号风险。

## 目标用户与核心场景
- **学生/自媒体/内容整理者**：把教程/资料视频离线保存
- **移动端用户**：手机浏览器直接保存到本地
- **批量整理**：一次性处理多条链接并查看进度
- **付费用户（Pro）**：更高并发、全平台 AI 总结，适合重度整理场景

## 当前已实现功能（截至最新版本）

### 下载核心
- **单链接 / 批量下载**：多行 URL 创建队列任务
- **实时进度**：SSE 推送下载进度（百分比 / 速度 / ETA）
- **临时存储与自动清理**：任务文件保留有限时间（TTL），到期删除
- **平台支持**：基于 `yt-dlp` 的通用解析；含抖音链接专项处理（见 `backend/app/douyin.py`）
- **cookies（默认关闭）**：管理员开关 + `X-Admin-Token` 后可上传 `cookies.txt`

### AI 视频总结
- 下载完成后自动总结（可配置开关）
- **免费用户**：默认仅 **B 站** 链接启用总结
- **Pro 用户**：**全平台** 链接均可总结
- 优先 B 站字幕提取，失败可回退 ASR（Whisper）
- 总结模板：`learning`（学习复盘）、`course`（课程精读）
- 导出：Markdown、Word、字幕文本

### 会员订阅（Stripe）
- **USD 订阅**：月付 / 年付，Stripe Checkout 托管支付
- **匿名设备识别**：Cookie `fvd_uid`，账单 SQLite 持久化
- **Pro 权益**：更高下载并发、全平台 AI 总结
- 当前阶段：**Test Mode 技术对接**（未上 Live、无合规主体页）

### 明确不做 / 默认关闭
- **完整账号体系**（注册登录、跨设备同步订阅）：暂无，以设备 Cookie 识别
- **cookies / 登录下载**：默认关闭
- **格式选择、字幕翻译、Video Q&A**：规划中，未实现

## 存储与架构约束
- **下载任务**：内存 `JobStore`（无任务历史持久化）
- **账单数据**：SQLite（`devices` / `subscriptions` / `stripe_events`）
- 不自研视频解析，统一 `yt-dlp` 封装

## 风险与约束
- **版权与平台风控**：UI 提示；默认关闭 cookies
- **资源与滥用**：单 IP 并发、单次 URL 数、TTL 清理；Pro 提高并发上限
- **支付安全**：Secret Key 仅服务端；Webhook 验签 + 事件幂等
