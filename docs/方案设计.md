## 总览
本项目采用 **FastAPI + Jinja（SSR）+ Tailwind** 的轻量架构，实现「万能视频下载」核心流程；下载能力统一封装 **yt-dlp**（不自研解析）。账单域单独使用 **SQLite + Stripe**，与下载任务的内存存储解耦。

### 设计目标
- **快速可用**：本地 / 公网一套代码
- **可扩展**：formats、字幕、总结、付费等能力可增量叠加
- **风险可控**：cookies 默认关闭、临时存储、限流与自动清理

## UI 设计（对齐参考站）
参考 `https://ai.codefather.cn/painting` 的关键风格点迁移到「下载」场景：
- **Hero 大标题**：黑色主标题 + 蓝色强调词
- **胶囊输入条**：圆角 pill + 图标 + 蓝色按钮
- **卡片瀑布流**：大圆角、柔和阴影、标签 chips
- **极简顶部栏**：汉堡菜单 / Logo + **开通 Pro** 入口

实现位置：
- 模板：`backend/app/templates/layout.html`、`index.html`、`billing_success.html`、`billing_cancel.html`
- 静态：`backend/app/static/app.js`、`app.css`

### 核心数据结构（下载域）
- **Job**：一条链接对应一个 Job（id、url、status、output_path、expires_at、ip、summary_*、meta）
- **Progress**：percent / speed / eta / filename / message
- **JobStatus**：`queued | downloading | finished | failed | expired`
- **SummaryStatus**：`not_requested | queued | transcribing | summarizing | done | failed | skipped`

### 核心数据结构（账单域 · SQLite）
- **devices**：`id`（= Cookie `fvd_uid`）、`stripe_customer_id`、`email`
- **subscriptions**：`stripe_subscription_id`、`status`、`price_id`、`current_period_end`
- **stripe_events**：已处理 `event.id`（Webhook 幂等）

## 任务执行模型
1. 创建任务 → `asyncio.create_task` 后台执行
2. 阻塞下载在 `ThreadPoolExecutor` 中运行 `yt-dlp`
3. `progress_hooks` 经 `loop.call_soon_threadsafe` 更新进度并 SSE 推送
4. 下载完成后（若 `enable_summary`）进入总结流水线（受 `FVD_MAX_SUMMARY_WORKERS` 限流）

## 用户识别与权益

### 设备 Cookie
- 中间件为每个浏览器签发 HttpOnly Cookie：`fvd_uid`（UUID）
- 首次访问写入，用于关联 Stripe 订阅与权益查询

### 权益矩阵
| 能力 | 免费用户 | Pro 用户（`active` / `trialing` 订阅） |
|------|----------|----------------------------------------|
| 同时下载任务数 / IP | `FVD_MAX_ACTIVE_JOBS_PER_IP`（默认 3） | `FVD_PRO_MAX_ACTIVE_JOBS_PER_IP`（默认 10） |
| AI 总结平台范围 | 仅 B 站（`FVD_SUMMARY_ONLY_BILIBILI=true` 时） | 全平台 |
| 订阅持久化 | — | SQLite `subscriptions` |

权益判定入口：`BillingService.get_entitlements(device_id)`，在 `POST /api/jobs` 创建任务时应用。

## cookies 能力（默认关闭）
- `FVD_ENABLE_COOKIES=false`（默认）
- 开启需配置 `FVD_ADMIN_TOKEN`，请求头 `X-Admin-Token`
- cookies 仅落盘到 job 临时目录，随 TTL 清理

## 临时存储与清理
- 下载目录：`{FVD_DOWNLOAD_ROOT}/{job_id}/`（默认 `/tmp/fvd/{job_id}/`）
- 账单库：`FVD_BILLING_DB_PATH`（默认 `/tmp/fvd/billing.db`）
- TTL：`FVD_TTL_SECONDS`（默认 3600）
- 清理：定时任务 `FVD_CLEANUP_INTERVAL_SECONDS`（默认 300）

## 运行模式

### 真实下载
- `FVD_DEMO_MODE=false`（默认）：`yt-dlp` 真实下载

### 演示 / CI
- `FVD_DEMO_MODE=true`：模拟进度与占位文件，无需外网
- `FVD_STRIPE_DEMO_MODE=true`：不调 Stripe API，本地 `/billing/demo-checkout` 模拟开通 Pro

## AI 视频总结

### 流程
1. 创建任务时根据权益设置 `meta.enable_summary`
2. 下载完成 → `transcribing`（字幕 / ASR）→ `summarizing`（DeepSeek）→ `done`
3. 落盘：`transcript.json`、`summary.json`；SSE 推送 `summary` 事件

### 结构化输出
- `outline`、`key_points`、`action_items`、`terms`、`model`、`template`

### 相关配置
- `FVD_ENABLE_AI_SUMMARY`、`FVD_SUMMARY_ONLY_BILIBILI`
- `FVD_DEEPSEEK_API_KEY`、`FVD_DEEPSEEK_BASE_URL`、`FVD_DEEPSEEK_MODEL`
- `FVD_MAX_SUMMARY_WORKERS`（默认 2）

### 失败策略
- 下载成功、总结失败：`summary_status=failed`，可 `retry`
- 未启用或非目标平台：`skipped`
- Demo 模式：生成演示转写与总结

## 会员订阅（Stripe）

### 售卖与履约
- **USD 订阅**（月付 / 年付），Stripe Checkout 托管页
- **履约以 Webhook 为准**：`checkout.session.completed`、`customer.subscription.updated/deleted`
- **幂等**：`stripe_events.event_id` 去重；创建 Session 支持 `idempotency_key`
- **安全**：`sk_*` / `whsec_*` 仅环境变量；Webhook 必须验签（不可先 `json()` 再验签）

### 本地联调（Test Mode + Stripe CLI）
1. Dashboard Test mode 创建 Product → 月付 / 年付 **Price**（`price_xxx`，非 `prod_xxx`）
2. 配置 `FVD_STRIPE_SECRET_KEY`（`sk_test_...`）、`FVD_STRIPE_PRICE_MONTHLY/YEARLY`
3. `stripe listen --forward-to http://127.0.0.1:8000/api/stripe/webhook` → `FVD_STRIPE_WEBHOOK_SECRET=whsec_...`
4. 测试卡 `4242 4242 4242 4242` → 确认 Webhook 与 `/billing/success`
